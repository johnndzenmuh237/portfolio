Customization Guide

Everything is customizable through CSS variables, HTML attributes, and a few lines of JavaScript. No framework knowledge required.

Colors

All colors are defined as CSS custom properties in the :root block of assets/css/style.css. Change them once and the entire template updates.

/* assets/css/style.css — :root */
:root {
  /* Backgrounds */
  --clr-bg:          #0C0C0F;   /* Main page background */
  --clr-surface:     #131317;   /* Cards, sidebar */
  --clr-surface-2:   #1A1A20;   /* Input fields, code blocks */
  --clr-border:      #242430;   /* All borders and dividers */

  /* Brand Colors — CHANGE THESE */
  --clr-accent:      #7B5EF6;   /* Primary: buttons, links, highlights */
  --clr-accent-2:    #5BE8C8;   /* Secondary: gradients, badges */
  --clr-accent-glow: rgba(123, 94, 246, 0.25); /* Glow shadow — match accent */

  /* Text */
  --clr-text:        #E8E8F0;   /* Body text */
  --clr-text-muted:  #7A7A90;   /* Secondary text, descriptions */
  --clr-text-faint:  #3E3E52;   /* Labels, timestamps */
  --clr-white:       #FFFFFF;   /* Headings */
}

Swap to a different accent color

To change the primary brand color, update --clr-accent and its corresponding glow:

/* Example: Electric Blue brand */
--clr-accent:      #3B82F6;
--clr-accent-2:    #06B6D4;
--clr-accent-glow: rgba(59, 130, 246, 0.25);

Fonts

The template uses three typefaces, all from Google Fonts:

VariableDefault FontUsage
--font-displaySyneAll headings (h1–h6), logo, stats
--font-bodyInterAll body text, paragraphs, buttons
--font-monoJetBrains MonoEyebrows, labels, code, badges

Change the display font

1. Update the Google Fonts <link> in each HTML file's <head>:

<!-- Replace Syne with Playfair Display -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?
  family=Playfair+Display:wght@400;700;800&
  family=Inter:wght@300;400;500;600&
  family=JetBrains+Mono:wght@400;500&display=swap" />

2. Update the variable in style.css:

--font-display: 'Playfair Display', serif;

Self-hosting fonts

For GDPR compliance or performance, place font files in assets/fonts/ and use @font-face in style.css instead of the Google Fonts import:

/* Remove the Google Fonts @import at the top of style.css, then add: */
@font-face {
  font-family: 'Syne';
  src: url('../fonts/Syne-Bold.woff2') format('woff2');
  font-weight: 700;
  font-display: swap;
}

<!-- index.html, about.html, etc. -->
<html lang="en" data-theme="ocean">

Adding a Theme Switcher Widget

Add this HTML anywhere in your page to show a visual swatch picker. The JS in darkmode.js handles it automatically:

<div class="theme-switcher">
  <button class="theme-swatch active" data-theme-value="default" aria-label="Default theme"></button>
  <button class="theme-swatch" data-theme-value="ocean"   aria-label="Ocean theme"></button>
  <button class="theme-swatch" data-theme-value="forest"  aria-label="Forest theme"></button>
  <button class="theme-swatch" data-theme-value="rose"    aria-label="Rose theme"></button>
  <button class="theme-swatch" data-theme-value="amber"   aria-label="Amber theme"></button>
  <button class="theme-swatch" data-theme-value="mono"    aria-label="Mono theme"></button>
</div>

Creating a Custom Theme

Add a new block to themes.css following the same pattern:

/* themes.css — add your own */
[data-theme="midnight"] {
  --clr-bg:          #0A0014;
  --clr-surface:     #120022;
  --clr-surface-2:   #1A0030;
  --clr-border:      #2C0050;
  --clr-accent:      #A855F7;
  --clr-accent-2:    #F0ABFC;
  --clr-accent-glow: rgba(168, 85, 247, 0.25);
  --clr-text:        #EDE9FE;
  --clr-text-muted:  #8B5CF6;
  --clr-text-faint:  #4C1D95;
  --clr-white:       #FFFFFF;
}

Dark / Light Mode

The default mode is dark. Light mode is applied when body has the class light-mode. The toggle in the navbar handles this automatically and persists via localStorage.

Default to light mode

To make light mode the default, add the class to <body> in every HTML file:

<body class="light-mode">

Disable dark mode toggle

Remove the toggle button from the navbar in each HTML file and comment out darkmode.js in the script tags. The site will stay in whatever mode <body> starts in.

Customise light mode colours

Light mode overrides live in assets/css/dark-mode.css under the body.light-mode selector:

/* dark-mode.css */
body.light-mode {
  --clr-bg:         #F7F7FA;
  --clr-surface:    #FFFFFF;
  --clr-accent:     #6648E8;  /* Slightly darker for contrast on white */
  /* ... */
}

Animations

Animations are powered by IntersectionObserver — no GSAP or external library needed. Any element with a data-animate attribute will animate in when it enters the viewport.

Animation variants

AttributeEffect
data-animate="fade-up"Fade in from below (most common)
data-animate="fade-down"Fade in from above
data-animate="fade-left"Slide in from left
data-animate="fade-right"Slide in from right
data-animate="fade-scale"Scale up from 92%
data-animate="fade"Opacity only, no transform

Stagger children

Add data-stagger to a parent element to automatically stagger the entrance animation of all direct children:

<div class="grid-3" data-stagger>
  <div class="card">...</div>
  <div class="card">...</div>  <!-- 80ms delay -->
  <div class="card">...</div>  <!-- 160ms delay -->
</div>

Delay individual elements

<h2 data-animate="fade-up">Title</h2>
<p  data-animate="fade-up" data-delay="200">Subtitle</p>
<div data-animate="fade-up" data-delay="400">Buttons</div>

Disable all animations

Remove animations.css from the <head> of each HTML file. The prefers-reduced-motion media query already disables animations automatically for users who have enabled that OS setting.

Layout & Spacing

All spacing, max-widths, and border radii are tokenised in style.css:

/* Spacing scale */
--sp-1: 0.25rem;  --sp-2: 0.5rem;   --sp-3: 0.75rem;
--sp-4: 1rem;     --sp-6: 1.5rem;   --sp-8: 2rem;
--sp-10: 2.5rem;  --sp-12: 3rem;    --sp-16: 4rem;
--sp-20: 5rem;    --sp-24: 6rem;    --sp-32: 8rem;

/* Max widths */
--max-w:    1200px;   /* Default container */
--max-w-sm:  760px;   /* Narrow container (.container--sm) */

/* Border radius */
--radius-sm: 6px;   --radius-md: 12px;
--radius-lg: 20px;  --radius-xl: 32px;
--radius-full: 9999px;

Section spacing

Use the .section class for standard vertical padding, or .section--sm for tighter spacing:

<section class="section">...</section>      <!-- 8rem top/bottom -->
<section class="section--sm">...</section>  <!-- 5rem top/bottom -->

Navigation

Adding / removing nav links

Edit the .nav-links and .nav-mobile sections in every HTML file. Remember to set the class="active" on the correct link for the current page:

<nav class="nav-links">
  <a href="index.html" class="active">Home</a>
  <a href="about.html">About</a>
  <a href="projects.html">Work</a>
  <a href="services.html">Services</a>
  <a href="blog.html">Blog</a>
  <a href="contact.html">Contact</a>
  <!-- Add more links here -->
</nav>

Adding New Sections

To add a new section, copy this boilerplate and paste it between existing sections in any HTML file:

<section class="section" id="your-section-id" aria-labelledby="your-heading-id">
  <div class="container">

    <!-- Optional: Section Header -->
    <div class="section-header" data-animate="fade-up">
      <div class="eyebrow">Category Label</div>
      <h2 id="your-heading-id">Section Title</h2>
      <p>Optional subtitle or description text.</p>
    </div>

    <!-- Your content here -->
    <div class="grid-3" data-stagger>
      <div class="card">
        <div class="card-body">
          <h3>Card Title</h3>
          <p>Card description.</p>
        </div>
      </div>
    </div>

  </div>
</section>

JavaScript Configuration

Counter animation

Any element with data-count will animate its number when scrolled into view:

<span data-count="120" data-suffix="+" data-duration="1800">120+</span>
<!-- data-count    = target number           -->
<!-- data-suffix   = appended string (e.g. +, %, K) -->
<!-- data-prefix   = prepended string (e.g. $) -->
<!-- data-decimals = decimal places (default 0) -->
<!-- data-duration = animation duration in ms  -->

Typed / rotating text

<strong data-typed="UI/UX Designer,Web Developer,Brand Strategist">
  UI/UX Designer
</strong>

Skill bars

<div class="skill-bar">
  <div class="skill-bar-fill" data-pct="85"></div>
</div>
<!-- data-pct = percentage (0–100). Animates on scroll. -->

Portfolio filter

Add data-category="web, branding" (comma-separated) to any .project-card. Filter buttons should have matching data-filter="web" values. The "all" filter always shows everything.

Lightbox

<a href="assets/images/projects/project-1-full.jpg"
   data-lightbox="assets/images/projects/project-1-full.jpg"
   data-caption="Optional caption text">
  <img src="assets/images/projects/project-1.jpg" alt="..." />
</a>

Parallax

<div data-parallax="0.3">
  <!-- Moves at 30% of scroll speed. Use values 0.1–0.5 for subtle effect. -->
</div>

Contact form endpoint

<!-- Formspree -->
<form id="contact-form" data-endpoint="https://formspree.io/f/XXXXXXXX">

<!-- Netlify -->
<form id="contact-form" name="contact" netlify data-netlify="true">

<!-- Custom backend (POST JSON) -->
<form id="contact-form" data-endpoint="https://api.yourdomain.com/contact">

Tip: All JavaScript is vanilla ES6 with no dependencies. You can open any .js file, read it top-to-bottom, and understand exactly what it does. Well-commented throughout.
