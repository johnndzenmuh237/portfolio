Installation Guide

Everything you need to get Portfolio Pro Template up and running in minutes.

v1.0.0 No Build Required Vanilla HTML/CSS/JS

Requirements

Portfolio Pro Template is a pure HTML/CSS/JavaScript template — no build tools, no frameworks, no Node.js required. All you need is:

  • A modern web browser (Chrome, Firefox, Safari, Edge)
  • A text editor (VS Code recommended)
  • A web server or hosting provider to deploy
  • Optional: VS Code Live Preview or Live Server extension for local development

Quick Start

Step 1 — Download & Unzip

Unzip the template into your projects folder. No npm install, no build step.

unzip portfolio-pro-template.zip -d my-portfolio
cd my-portfolio

Step 2 — Add Your Images

Place your images in the correct folders under assets/images/. See the Assets Guide for exact filenames and recommended dimensions.

Step 3 — Personalise the Content

Open each HTML file in your text editor and replace all placeholder text marked with square brackets:

<!-- Find and replace these placeholders: -->
[Your Name]          → Your actual name
[YN]                 → Your initials (logo text)
[Your City, Country] → e.g. London, UK
hello@yourdomain.com → Your email address
yourdomain.com       → Your website URL
yourusername         → Your social media handles

VS Code Tip: Use Ctrl+Shift+H (Find & Replace across files) to replace all placeholders at once.

Step 4 — Customise Colours & Fonts

Open assets/css/style.css and edit the CSS custom properties in the :root block. See the Customization Guide for details.

Step 5 — Connect Your Contact Form

The contact form works out of the box in "demo mode" (simulates success). To connect it to a real backend, add your endpoint URL to the form's data-endpoint attribute:

<!-- Formspree (free tier available) -->
<form id="contact-form" data-endpoint="https://formspree.io/f/YOUR_FORM_ID">

<!-- Netlify Forms (automatic when deployed to Netlify) -->
<form id="contact-form" name="contact" netlify>

Local Development

Because the template uses ES modules and fetch, you need to serve it over HTTP (not file://).

Option A — VS Code Live Server (Recommended)

  1. Install the Live Server extension
  2. Open the template folder in VS Code
  3. Right-click index.html → "Open with Live Server"

Option B — Python HTTP Server

# Python 3
python -m http.server 3000
# Then visit: http://localhost:3000

Option C — Node.js serve

npx serve .
# Then visit the URL shown in your terminal

Custom Domain

Update the canonical URL and Open Graph meta tags in every HTML file's <head> to reflect your actual domain before going live:

<link rel="canonical" href="https://yourdomain.com" />
<meta property="og:url" content="https://yourdomain.com" />

Troubleshooting

Images not showing

Check that your image filenames exactly match the src paths in the HTML. Image paths are case-sensitive on Linux servers. See the Assets Guide for the full list of expected filenames.

Animations not working

The JavaScript must be loaded and the page served over HTTP. Animations use IntersectionObserver — if you're opening the HTML directly as a file (file://), some features may not work. Use a local server (see above).

Dark mode not persisting

The theme preference is saved to localStorage. Ensure the page is served over HTTP and that you haven't disabled localStorage in your browser settings.

Contact form sending to nowhere

By default the form runs in demo mode. Add your data-endpoint attribute (see Step 5 above) to connect it to a real backend.
